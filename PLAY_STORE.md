# Google Play Store — Build & Publish Guide

## Prerequisites

1. **Android Studio** — download from https://developer.android.com/studio
2. **Java 17+** — Android Studio bundles this
3. **Google Play Developer account** — $25 one-time fee at https://play.google.com/console

## Quick Start

```bash
cd client

# Set your production server URL (REQUIRED for Android builds)
# Edit .env.capacitor and set VITE_API_URL to your Railway URL:
#   VITE_API_URL=https://your-app.up.railway.app

# Build, sync, and open in Android Studio — one command:
npm run android
```

This runs `vite build` → `cap sync android` → opens Android Studio.

## Step-by-Step

### 1. Set Production API URL

Edit `client/.env.capacitor`:
```
VITE_API_URL=https://your-app.up.railway.app
```

Then build with that env:
```bash
cp .env.capacitor .env.production.local
npm run build
npx cap sync android
```

### 2. Open in Android Studio

```bash
npx cap open android
```

### 3. Generate Signed APK/AAB

In Android Studio:
1. **Build → Generate Signed Bundle / APK**
2. Choose **Android App Bundle** (AAB) — required for Play Store
3. Create a new keystore (first time only):
   - Key store path: choose a safe location (NOT in the repo)
   - Password: pick a strong password — **save this, you need it forever**
   - Key alias: `realsynth`
   - Key password: same or different
   - Fill in certificate info (name, org, etc.)
4. Select **release** build variant
5. Click **Finish** — AAB will be at `android/app/release/app-release.aab`

> **IMPORTANT:** Back up your keystore file and passwords. If you lose them, you can never update the app on Play Store.

### 4. Test Locally (Debug APK)

```bash
# Build debug APK without signing
npx cap run android
```

Or in Android Studio: **Run → Run 'app'** with an emulator or connected device.

### 5. Play Store Submission

#### Create App Listing
1. Go to https://play.google.com/console
2. **Create app** → name: "Real Synth"
3. Fill in:
   - **App category**: Social / Communication
   - **Content rating**: Complete the IARC questionnaire — answer YES to:
     - "Does the app contain user-generated content?" → Yes
     - "Does it reference sexuality, violence, or mature themes?" → Yes
     - Target rating: **Mature 17+**
   - **Target audience**: 17+ (NOT children)
   - **AI disclosure**: In the app description, include: "All conversations in Real Synth are generated by artificial intelligence."

#### Screenshots
- Take 4-8 screenshots at phone resolution (1080×1920 or similar)
- Must include: landing screen, chat conversation, brain panel
- Use Android Studio emulator or a real device

#### Privacy Policy (required)
You need a privacy policy URL. Minimum content:
- What data is collected (messages, phrase hash)
- Where it's stored (MongoDB Atlas)
- No data is sold to third parties
- How to request data deletion (email jacesabr@gmail.com)

#### Upload

2. Upload the `.aab` file
3. Fill release notes
4. **Review and roll out**

## Updating the App

After making code changes:
```bash
cd client
npm run build        # rebuild web assets
npx cap sync android # copy to Android project
npx cap open android # open in Android Studio
# Then: Build → Generate Signed Bundle/APK → release
```

Bump `versionCode` and `versionName` in `android/app/build.gradle` for each update.
