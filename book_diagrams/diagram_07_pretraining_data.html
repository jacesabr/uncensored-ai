Scraped web pages. Filtered via quality heuristics, perplexity scoring, dedup. Highest volume, lowest quality per token.
Books / Long-form Text

Code

Wikipedia
5–10%

Scientific / Academic

Training data composition is a critical hyperparameter. Code data dramatically improves logical reasoning beyond programming tasks. Quality filtering (dedup, perplexity scoring, classifier-based) is as important as quantity — Chinchilla showed that 1.4T high-quality tokens trained a 70B model to match GPT-3's 175B on 300B tokens. Oversampling high-quality sources (Wikipedia, books) by 2–4× is standard practice.
Python Demo — stdlib only
import random, collections

# Simulate a weighted data mixer
sources = {
    "web":        (0.45, 1.0),   # (sampling weight, quality score 0-1)
    "books":      (0.17, 0.95),
    "code":       (0.17, 0.90),
    "scientific": (0.10, 0.92),
    "wikipedia":  (0.06, 0.98),
    "other":      (0.05, 0.70),
}

def sample_batch(sources, n=1000):
    names = list(sources.keys())
    weights = [sources[s][0] for s in names]
    total = sum(weights)
    normed = [w/total for w in weights]
    counts = collections.Counter()
    for _ in range(n):
        r = random.random(); cum = 0.0
        for name, p in zip(names, normed):
            cum += p
            if r <= cum: counts[name] += 1; break
    return counts

counts = sample_batch(sources, 10000)
avg_quality = sum(counts[s]*sources[s][1] for s in sources) / 10000

print(f"Sampled 10,000 tokens from mixture:")
for src, cnt in sorted(counts.items(), key=lambda x: -x[1]):
    bar = "█" * (cnt // 100)
    print(f"  {src:12s} {bar} {cnt:5d} ({cnt/100:.1f}%)")
print(f"\nWeighted avg quality score: {avg_quality:.3f}")
# Try boosting "code" weight to 0.30 and reducing "web" — see how avg quality shifts.
