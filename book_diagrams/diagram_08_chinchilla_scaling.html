Chinchilla Rule
N_tokens ≈ 20 × N_params
For compute-optimal training, train a model for 20 tokens per parameter. A 7B model → 140B tokens minimum.

Green = near compute-optimal. Orange = undertrained. Numbers are approximate cross-entropy loss values.
The Chinchilla paper (Hoffmann et al., 2022) found that for a fixed compute budget C ≈ 6ND FLOPs, the optimal allocation is N_params ∝ C^0.5 and D_tokens ∝ C^0.5, yielding the 20× rule. Post-Chinchilla models like LLaMA-3 deliberately "over-train" (beyond compute-optimal) to produce smaller models that are cheap to serve at inference time while matching larger undertrained models.
Python Demo — stdlib only
import math

# Chinchilla scaling law: approximate loss function
# L(N, D) ≈ E + A/N^α + B/D^β  (simplified Kaplan/Hoffmann fit)
E = 1.61   # irreducible entropy of natural language
A = 406.4
B = 410.7
alpha = 0.34
beta  = 0.28

def loss(N, D):
    """Approximate cross-entropy loss for N-param model trained on D tokens."""
    return E + A / (N ** alpha) + B / (D ** beta)

def optimal_N_D(C, tokens_per_param=20):
    """Given compute budget C FLOPs, find optimal N and D."""
    # C ≈ 6*N*D  →  N = C / (6*D)  with D = tokens_per_param * N
    N = math.sqrt(C / (6 * tokens_per_param))
    D = tokens_per_param * N
    return int(N), int(D)

print("Compute budget → optimal (N, D) → predicted loss:")
budgets = [("GPT-3 budget", 3.14e23), ("LLaMA-2-7B", 2e23), ("LLaMA-3-8B", 1.5e24)]
for name, C in budgets:
    N, D = optimal_N_D(C)
    l = loss(N, D)
    print(f"  {name}: N={N/1e9:.1f}B params, D={D/1e12:.1f}T tokens → loss≈{l:.3f}")

print("\nActual GPT-3 (175B, 300B tokens) vs optimal:")
gpt3_actual = loss(175e9, 300e9)
gpt3_opt_N, gpt3_opt_D = optimal_N_D(3.14e23)
gpt3_optimal = loss(gpt3_opt_N, gpt3_opt_D)
print(f"  Actual:  {gpt3_actual:.3f}")
print(f"  Optimal: {gpt3_optimal:.3f}  (saves {gpt3_actual-gpt3_optimal:.3f} loss units)")
