Compute cost comparison
Without KV cache: step t costs O(t²) — recompute ALL attention for ALL tokens
With KV cache: step t costs O(t) — only compute Q for new token, dot with cached K

Memory formula: bytes = seq_len × d_model × 2(K+V) × n_layers × 2(fp16) bytes
LLaMA-3-8B: 128K × 4096 × 2 × 32 × 2 = ~64GB at full context (why GQA matters!)
Benefit
Each generated token takes constant time (ignoring attention cost). Batching multiple sequences amortizes memory. Prefill parallelizes across full sequence.
Cost
Memory grows linearly. At 128K context, 8B model needs 64GB+ for KV alone. GQA (Grouped-Query Attention) reduces this by sharing K/V across multiple Q heads.
GQA/MQA fix
LLaMA-3 uses GQA: 8 Q-heads share 1 K/V-head group. 8× memory reduction at <1% quality loss. Multi-Query Attention (MQA) takes this further: 1 K/V total.
The KV cache is the key inference optimization in transformer LLMs. During prefill, all K and V vectors for the prompt are computed and stored. During decoding, only the new token's Q, K, V are computed — K and V appended to cache, Q used to attend over all cached K's. This reduces per-token decode cost from O(n²) to O(n), making long-context generation practical.
Python Demo — stdlib only
import math, random

def dot(a,b): return sum(x*y for x,y in zip(a,b))
def softmax(v):
    m=max(v); e=[math.exp(x-m) for x in v]; t=sum(e)
    return [x/t for x in e]

class KVCache:
    def __init__(self): self.K = []; self.V = []
    def append(self, k, v): self.K.append(k); self.V.append(v)
    def size_bytes(self, dtype_bytes=2):  # fp16
        return len(self.K) * len(self.K[0]) * 2 * dtype_bytes  # K + V

d_k = 16  # try 64 or 128
n_layers = 4
cache = KVCache()

# Simulate autoregressive decode with KV cache
print(f"{'Step':5s} {'CacheSize':10s} {'MemKB':8s} {'AttentionOps':12s} {'Without cache'}")
for step in range(1, 12):
    # New token: compute Q, K, V (O(d) each)
    q = [random.gauss(0,1) for _ in range(d_k)]
    k = [random.gauss(0,1) for _ in range(d_k)]
    v = [random.gauss(0,1) for _ in range(d_k)]
    cache.append(k, v)

    # Attention: Q dot-product with all cached K's (O(t) ops)
    scores = softmax([dot(q, ki)/math.sqrt(d_k) for ki in cache.K])
    out = [sum(scores[j]*cache.V[j][i] for j in range(len(cache.K))) for i in range(d_k)]

    mem_kb = cache.size_bytes() * n_layers / 1024
    ops_with = step      # O(t) with cache
    ops_without = step**2  # O(t²) without cache
    print(f"{step:5d} {step:10d} {mem_kb:8.2f} {ops_with:12d} {ops_without:13d}")
