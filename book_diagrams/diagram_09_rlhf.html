PPO Objective with KL Penalty
max E[ r(x,y) − β · KL(π_RL ‖ π_SFT) ]
Why KL penalty? Without it, the policy "reward hacks" — learning to produce responses that score high on the reward model but are bizarre or incoherent. The KL term keeps the RL policy close to the SFT policy, preventing mode collapse. β ≈ 0.02–0.05 is typical. Higher β = stays closer to SFT, lower β = more aggressively optimizes reward.
Modern variants
DPO (Direct Preference Optimization) skips the RM entirely, converting pairwise preferences to a classification loss directly on the policy. Used by LLaMA-3, Mistral.
Data requirements
InstructGPT used ~13K SFT examples, ~33K RM comparison pairs. Quality >> quantity. Annotators must follow detailed rubrics (helpfulness, harmlessness, honesty).
Constitutional AI
Anthropic's variant: model critiques its own outputs using a "constitution" of principles, then revises. Reduces need for human feedback in Stage 1/2 (RLAIF).
RLHF (Ouyang et al. 2022, InstructGPT) transformed ChatGPT from a text completer to an instruction follower. The three-stage pipeline — SFT → reward model → PPO — bridges the gap between "predict the next token" and "do what humans actually want." The KL penalty is critical: pure reward maximization leads to degenerate outputs.
Python Demo — stdlib only
import math, random

# Simulate reward model training on pairwise preferences
def sigmoid(x): return 1 / (1 + math.exp(-x))

# Pairwise preference data: (reward_chosen, reward_rejected)
def rm_loss(r_chosen, r_rejected):
    """Bradley-Terry pairwise loss: -log σ(r_chosen - r_rejected)"""
    return -math.log(sigmoid(r_chosen - r_rejected) + 1e-9)

# PPO-style objective (simplified scalar version)
def ppo_objective(reward, kl_penalty, beta=0.03):
    """Reward minus KL divergence penalty."""
    return reward - beta * kl_penalty

# Simulate training loop
rm_score = 0.5   # current reward model scalar (starts naive)
sft_score = 0.5  # reference policy
beta = 0.03       # KL penalty weight — try 0.0 or 0.3

print(f"{'Step':5s} {'RL reward':12s} {'KL penalty':12s} {'Objective':12s} {'Status'}")
for step in range(10):
    # Simulate RL reward improving but KL drifting
    rm_score += random.uniform(0.1, 0.3)
    kl = (rm_score - sft_score) ** 2 * 5  # KL grows as we diverge from SFT
    obj = ppo_objective(rm_score, kl, beta)
    status = "✓ good" if obj > 1.5 else ("⚠ drifting" if kl > 2.0 else "→ training")
    print(f"{step+1:5d} {rm_score:12.3f} {kl:12.3f} {obj:12.3f} {status}")

print(f"\nRM pairwise loss examples:")
for (rc, rr) in [(3.0, 1.0), (1.2, 1.1), (5.0, 0.5)]:
    print(f"  chosen={rc:.1f}, rejected={rr:.1f} → loss={rm_loss(rc,rr):.4f}")
# Try beta=0.0 (no KL penalty) — watch objective keep climbing but model drifts.
# Try beta=0.3 (heavy penalty) — objective stays low, model stays close to SFT.
